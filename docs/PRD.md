# 山資訊系統 PRD (產品需求文件)

## 1. 專案概述
### 背景與動機
隨著戶外活動的盛行，越來越多人參與登山活動。然而，台灣的山岳資訊經常散落在不同的網站與平台，且資訊可能不夠即時或完整。為了提升登山安全並提供更友善的資訊查詢體驗，我們決定開發「山資訊」系統，整合台灣各大山岳的重要資訊，讓使用者能一站式獲取所需內容。

### 目標用戶
- 想爬山的人：包含初學者到進階登山客，需要查詢路線、高度、安全狀況及裝備建議的使用者。

### 核心價值主張
提供一個統整、清晰、即時的台灣山岳資訊平台，結合安全警示、裝備清單與山友社群回饋，降低登山門檻並提升活動安全性。

## 2. 功能需求
以下為系統主要提供的 5 項核心功能：

### 2.1 山域檢索
- **作為** 想爬山的人，**我希望** 能夠透過關鍵字或分類（如難度、地區）搜尋山域，**以便** 快速找到我想去的山岳資訊（包含海拔、位置、開放時間等）。

### 2.2 安全警示指標
- **作為** 想爬山的人，**我希望** 能在山岳資訊頁面看到明確的安全警示指標（如氣候異常、步道崩塌、管制資訊），**以便** 評估當下是否適合前往，確保自身安全。

### 2.3 數位化裝備清單與注意事項
- **作為** 想爬山的人，**我希望** 系統能針對不同的山岳或路線提供推薦的數位裝備清單與專屬注意事項，**以便** 我能事前做好充分準備，避免遺漏重要物品。

### 2.4 路況時間軸
- **作為** 想爬山的人，**我希望** 能夠查看路線沿途的地標與預估耗時形成的路況時間軸，**以便** 我能掌握行程進度與各個路段的地形或難度變化。

### 2.5 評論與分享區
- **作為** 想爬山的人，**我希望** 可以在個別山資訊欄的下方閱讀或發表評論與近期照片，**以便** 參考其他山友的最新回報或分享自己的心得。

## 3. 非功能需求
### 技術限制
- 系統建置採用 Flask + Jinja2 作為後端與樣板引擎。
- 資料庫使用 SQLite，以便快速開發與部署。

### 效能與安全考量
- **效能**：頁面載入速度需在合理時間內完成（建議 2 秒內），特別是圖片（山友分享）需有基本的壓縮或分頁處理。
- **安全**：
  - 留言區須具備基本的防 SQL 注入 (SQL Injection) 與跨網站指令碼 (XSS) 的防護。
  - 用戶密碼需經過加密（如 bcrypt）儲存。

## 4. MVP 範圍 (最小可行性產品)
### Must Have (必須有)
- 山岳基本資料庫（海拔、位置、簡介）
- 山域檢索功能（文字搜尋）
- 安全警示指標（靜態資訊提示）
- 簡單的評論區（允許用戶留言）

### Should Have (應該有)
- 數位化裝備清單與注意事項展示
- 路況時間軸（圖文或列表呈現）
- 基礎的會員系統（註冊/登入後才能評論）

### Nice to Have (可以有)
- 社群分享功能（一鍵分享到社群媒體）
- 照片上傳功能（評論中附帶照片）
- 使用者收藏/打卡功能
